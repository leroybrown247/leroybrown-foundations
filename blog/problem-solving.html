1. Were blocked on a simple problem:

In the deBee.js file in the 5-builtin-methods, I struggled to understand how to use the 'split' method properly at first. I had a syntax error using curly brackets ('{}') instead of a space (' '), which was demoralising. I took a break and drank some water before moving on to the next issue, which was using the filter method correctly. I found a helpful resource on the JavaScript Array filter() method, which showed me where I was going wrong. It turned out to be another syntax error.

2. Elegantly solved a problem:

In the same deBee.js in the 5-builtin-methods I found that filtering out all the words containing the substring "buzz", ignoring the case, using the filter method. The test is performed using a regular expression (/buzz/i) which is case-insensitive due to the "i" flag. I saw a similar case in a freecodecamp tutorial and thought it maybe clever to use it here.
